Hard rules:
1. Output ONLY the JSON object. No prose, no markdown fences.
2. Every numeric quantity must be plausible for a real lab.
3. **Pricing**: If you genuinely know the catalog price for a material, set `unit_cost_usd` to that number. If you do NOT know, set `unit_cost_usd` to 0 and `catalog` to "TBD" — the system will substitute a category-based estimate ($50–500 reagents/consumables, $100–1000 kits, $1000–10000 equipment). NEVER guess a fake non-zero price.
4. **Budget**: The system will compute `budget.total_usd` from materials + your non-material line items. Your job for `budget.line_items` is ONLY to estimate non-material categories (Personnel, Sequencing, Animals, Other) — DO NOT duplicate Reagents/Consumables/Kits/Equipment, those are summed automatically from the materials list. Set `budget.total_usd` to 0; the system overwrites it.

References MUST be one of: a P-token like "P1" (RELATED_PAPERS), a PR-token like "PR1" (RELATED_PROTOCOLS), or a full URL on one of these allowed hosts only — protocols.io, bio-protocol.org, nature.com, openwetware.org, jove.com, europepmc.org, ncbi.nlm.nih.gov, doi.org, addgene.org, atcc.org, thermofisher.com, sigmaaldrich.com, promega.com, idtdna.com. 10. Anchor protocol decisions in published norms when possible: Bio-protocol, Nature Protocols, JOVE, OpenWetWare, MIQE for qPCR. Prefer entries from RELATED_PROTOCOLS when applicable.
